差速移动机器人（Differential Drive Robot）的运动学模型建立方法：
### 1. 基本假设与定义
在建立模型前，通常基于以下物理假设：
*   **纯滚动无滑动**：轮子与地面接触点速度为零，不打滑。
*   **非完整约束**：机器人只能沿车身纵轴方向运动，**不能横向侧滑**（即 $v_x = 0$ 在机器人坐标系下）。
*   **定义变量**：
    *   $W$：两轮之间的间距（轮距）。
    *   $v_{left}, v_{right}$：左右轮的线速度。
    *   $(x, y)$：机器人在大地坐标系下的位置。
    *   $\theta$ (或 $\psi$)：机器人的航向角（与X轴或Y轴的夹角）。
### 2. 局部坐标系下的速度解算（逆运动学基础）
首先计算机器人中心点的**线速度 $v$** 和 **角速度 $\omega$**。
根据几何关系（参考图1）：
*   **前进速度**是左右轮速度的平均值：
    $$ v = \frac{v_{right} + v_{left}}{2} $$
*   **旋转角速度**取决于左右轮的速度差和轮距：
    $$ \omega = \dot{\theta} = \frac{v_{right} - v_{left}}{W} $$

### 3. 全局坐标系下的位姿更新（运动学方程）
将局部速度投影到全局地图坐标系（$X_{ground}, Y_{ground}$）中。
利用三角函数关系：
*   $\dot{x} = v \cdot \cos\theta$ （X轴方向分速度）
*   $\dot{y} = v \cdot \sin\theta$ （Y轴方向分速度）
*   $\dot{\theta} = \omega$ （角度变化率）

**综合矩阵形式（状态空间方程）：**
$$
\begin{bmatrix} \dot{x} \\ \dot{y} \\ \dot{\theta} \end{bmatrix} = 
\begin{bmatrix} \cos\theta & 0 \\ \sin\theta & 0 \\ 0 & 1 \end{bmatrix} 
\begin{bmatrix} v \\ \omega \end{bmatrix}
$$
### 4. 模型的离散化（计算机实现）
由于计算机只能处理离散的时间步长（$\Delta t$ 或 $T$），需要将上述微分方程转化为差分方程：
*   $x(k+1) = x(k) + \dot{x} \cdot T$
*   $y(k+1) = y(k) + \dot{y} \cdot T$
*   $\theta(k+1) = \theta(k) + \dot{\theta} \cdot T$

### 5.极坐标模型（用于控制）
为了方便设计控制器（如让机器人走到某一点），会将直角坐标 $(x, y, \theta)$ 转换为极坐标 $(\rho, \alpha, \beta)$：
*   $\rho$：机器人到目标的距离。
*   $\alpha$：机器人朝向与目标连线的夹角。
*   $\beta$：目标连线与目标朝向的夹角。

# AGV（自动导引车）设计思路

设计一台实用的AGV需要**系统工程思维**。
## 一、需求分析与场景定义（设计起点）

| 问题 | 设计影响 |
|------|---------|
| **载重多少？** | 电机扭矩、车架强度、轮子选型 |
| **运行速度？** | 电机转速、控制频率、安全制动距离 |
| **导航精度要求？** | 定位方案（二维码/激光/视觉） |
| **工作环境？** | 室内/室外、地面平整度、光照条件 |
| **续航要求？** | 电池容量、是否支持自动充电 |
| **通信需求？** | WiFi/5G、调度系统对接协议 |

## 二、机械结构设计

### 1. 底盘选型
```
常见驱动方式对比：
┌─────────────┬─────────────┬─────────────┐
│ 差速驱动    │ 阿克曼转向  │ 全向轮      │
│ (2轮+万向轮)│ (汽车式)    │ (麦克纳姆轮)│
├─────────────┼─────────────┼─────────────┤

│ ❌ 地面要求高│ ❌ 不能横移 │ ❌ 效率略低  │
└─────────────┴─────────────┴─────────────┘
```

## 三、驱动与运动控制（核心）

### 1. 控制架构
```
上层调度 → 路径规划 → 运动控制器 → 电机驱动器 → 电机
                              ↑
                         编码器反馈（闭环）
```

### 2. 运动学模型应用
利用之前推导的差速模型：
```python
# 伪代码：轮速解算（逆运动学）
def compute_wheel_speeds(v_desired, omega_desired, wheel_base_W):
    v_right = v_desired + (wheel_base_W * omega_desired) / 2
    v_left  = v_desired - (wheel_base_W * omega_desired) / 2
    return v_left, v_right
```

### 3. 闭环控制
- **速度环**：PID控制，让电机实际转速跟踪指令
- **位置环**：结合编码器+定位系统，修正累积误差
- **状态估计**：可用卡尔曼滤波融合多传感器数据


## 四、导航与定位系统（眼睛）

### 方案对比
| 方案 | 精度 | 成本 | 适用场景 |
|------|------|------|---------|
| **磁条/磁钉** | ±10mm | 低 | 固定路径、简单场景 |
| **二维码** | ±5mm | 中 | 仓储、路径可规划 |
| **激光SLAM** | ±20mm | 高 | 动态环境、柔性路径 |
| **视觉SLAM** | ±30mm | 中高 | 光照稳定、特征丰富 |
| **UWB/RTK** | ±10cm | 中 | 室外/大范围定位 |

## 五、感知与安全系统（保命）

### 1. 障碍物检测
- **前方**：2D激光雷达 / 深度相机（检测动态障碍）
- **四周**：超声波/红外（近距离防撞）
- **底部**：防跌落传感器（台阶检测）

### 2. 安全策略（分级）
```
🟢 正常：全速运行
🟡 预警：检测到障碍→减速→声光提示
🔴 急停：碰撞/急停按钮→硬件切断电机供电
```
